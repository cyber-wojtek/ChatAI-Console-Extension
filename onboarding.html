✦ ChatAI Console
OAuth Bridge · Setup

What this extension does

This extension enables Google sign-in for Claude.ai and MiniApps.ai accounts in ChatAI Console, which runs locally on your machine at http://localhost:5000.

Google's sign-in system only issues tokens for registered domains. This extension runs on those domains so the sign-in happens from the right origin, then sends the resulting token or auth code to your local Console server — never to any external server.

Data transmitted

When you initiate a Google sign-in from ChatAI Console:

Claude sign-in A one-time Google OAuth authorization code is sent to http://localhost:5000. This code is used by your local Console to authenticate with Claude.ai on your behalf.
MiniApps sign-in A Google ID token (JWT) issued by Google's Identity Services is sent to http://localhost:5000. This token is used by your local Console to authenticate with MiniApps.ai on your behalf.
✓ No data is sent to any external server — only to localhost:5000 on your own machine.

What is NOT collected

Browsing activityNever collected or transmitted.

When does it activate?

The extension is disabled by default and does nothing during normal browsing. You must explicitly enable it from the extension popup before each sign-in, then disable it again when done if you wish. The content scripts only act when your local Console server confirms it initiated the current sign-in flow.

Your choice

To use Google sign-in with ChatAI Console, the extension needs to send your Google auth code or ID token to localhost:5000 (your local machine only) when you initiate a sign-in.

You can decline and still use ChatAI Console by entering session keys manually. You can change this choice any time from the extension popup.

Declining does not uninstall the extension. The bridge remains inactive until you enable it from the popup.

You can change this in the extension popup at any time.
